# 慧农项目 Vibe Coding 核心约束
## 1. 技术栈边界（不可变更）
- 前端：Flutter 3.24+（移动端），严格遵循 null-safe 规范
- 网络层：必须基于 Dio 5.0+ 封装（单例模式、拦截器、通用请求方法）
- 后端对接：已有的 Python FastAPI 接口（用户/问诊/首页模块），所有请求需对齐后端数据模型
- 状态管理：轻量 Provider，避免过度封装
- 样式体系：两套配置（默认=Google 原生简洁大气；适老化=符合国家适老化标准）

## 2. 架构设计要求（解耦+高性能）
- 代码分层：api/（Dio 封装+接口分类）→ models/（数据模型）→ pages/（页面）→ utils/（工具类）→ provider/（全局状态）
- 异步处理：所有网络请求、本地存储、样式切换必须异步执行，避免阻塞 UI
- 扩展原则：新增功能时需复用已有封装（如 Dio 实例、通用组件），不重复造轮子

## 3. 样式核心规则
### 3.1 默认样式（简洁大气）
- 基础：Google 原生 Material Design，无冗余装饰，配色以白色+浅灰为主，强调色用绿色（#2E7D32）
- 交互：极简动效（仅点击反馈），布局间距统一，适配 4.7-6.7 英寸移动端
### 3.2 适老化样式（国家规范）
- 字体：标题≥20px，正文≥18px，按钮文字≥19px，无细体/斜体
- 配色：高对比度（文字黑/深灰，背景浅米，避免浅色文字+浅色背景）
- 交互：按钮尺寸放大（高度≥50px）、间距加宽（≥20px）、无滑动验证/复杂动画
- 切换逻辑：通过全局按钮触发，样式切换需实时生效，无需重启应用

## 4. 开发优先级
1. 先完成基础框架（Dio 封装+模型定义+路由）→ 2. 核心页面（首页/问诊/我的）→ 3. 样式体系 → 4. 适老化切换功能 → 5. 性能优化

# 新增：后端接口对接规则（必遵守）
## 1. 接口基础信息
- 后端基础URL：http://127.0.0.1:8000/api/v1
- 数据格式：所有请求/响应为 JSON，表单提交（如登录）用 FormData 格式
- 认证规则：登录接口返回 user_id/token，后续接口需在请求头携带 user_id
- 图片上传：接口接收 multipart/form-data 格式，文件字段名固定为 "file"

## 2. 核心接口映射
| 前端功能       | 后端接口                | 请求方式 | 关键参数                | 响应核心字段               |
|----------------|-------------------------|----------|-------------------------|----------------------------|
| 用户登录       | /users/login            | POST     | username, password      | id, username, status       |
| 图片上传识别   | /crops/identify         | POST     | user_id, file（图片）   | disease_name, advice, confidence |
| 首页数据获取   | /news/home              | GET      | -                       | banner_list, news_list     |
| 我的页面信息   | /users/info/{user_id}   | GET      | user_id（路径参数）     | username, phone, history   |

## 3. 对接要求
- 所有接口调用必须基于已封装的 Dio 单例，新增接口需归类到对应 api_service 文件
- 响应数据必须解析为 null-safe 的 Model 类，禁止直接使用 Map 解析
- 交互逻辑：请求时显示加载中，失败时弹出中文提示，成功后跳转/更新页面
